# Version 1.0 Phase 1: Canonical Endpoint Map

## Purpose
This map defines the approved endpoint surface for default runtime integration.

## Canonical families
- Platform: `/api/v1/*`
- Inventory: `/api/inventory/v1/*`

## Transitional (Deprecated In OpenAPI)
- Family: `/api/v1/inventory/*`
- Status: transitional compatibility aliases for inventory operations.
- OpenAPI metadata:
  - `deprecated: true`
  - `x-contract-status: transitional`
  - `x-deprecation-reason: Transitional compatibility aliases. Use /api/inventory/v1/* endpoints.`
- Known duplicate alias pair:
  - `GET /api/v1/inventory/{variantId}`
  - `GET /api/v1/inventory/variants/{variantId}`

## Explicitly non-canonical endpoint families
- `/api/storefront/v1/*` (legacy-storefront profile only)
- `/commerce/api/v1/*` and other `com.noura.platform.commerce.*` web APIs not used by active apps
