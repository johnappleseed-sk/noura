# Version 1.0 Phase 1: Contract Catalog

## Purpose
This document is the execution artifact for Version 1.0 Phase 1. It freezes the canonical runtime contract used by the current backend, admin dashboard, and storefront.

## Evidence baseline
- `backend/src/main/java/com/noura/platform/EnterpriseCommerceApiApplication.java`
- `backend/src/main/java/com/noura/platform/controller/*`
- `backend/src/main/java/com/noura/platform/inventory/api/*`
- `backend/src/main/resources/application.yml`
- `backend/src/main/resources/application-local-mysql.yml`
- `docs/runtime-canonical-contract-v1.md`

## Canonical runtime boundaries

### 1) Canonical API families
- `platform-commerce`: `/api/v1/*`
- `inventory-ops`: `/api/inventory/v1/*`

### 2) Canonical UI consumers
- `frontend/admin-dashboard` consumes both `/api/v1/*` and `/api/inventory/v1/*`.
- `frontend/storefront-noura` consumes `/api/v1/*` only.

### 3) Canonical backend module scope
- Active app package root: `com.noura.platform`.
- Excluded from default component scan: `com.noura.platform.commerce.*`.
- Active persistence units:
  - platform default unit: `com.noura.platform.domain.entity`
  - inventory unit: `com.noura.platform.inventory.domain` (separate datasource + Flyway location)

### 4) Authentication and authorization contract
- Auth entrypoints:
  - `POST /api/v1/auth/register`
  - `POST /api/v1/auth/login`
  - `POST /api/v1/auth/refresh`
- Admin capabilities contract:
  - `GET /api/v1/admin/capabilities`
  - capability map is source of truth for admin route and action gating.

### 5) Response contract
- Canonical envelope for active runtime: `ApiResponse` shape used in `/api/v1/*` and `/api/inventory/v1/*`.
- Canonical paginated shape: `PageResponse`.

## Non-canonical runtime boundaries
- Legacy storefront contract:
  - `/api/storefront/v1/*`
  - Profile-gated and non-canonical for default runtime.
- `com.noura.platform.commerce.*` domain APIs:
  - present in codebase but non-canonical for default runtime.

## Profile and schema policy matrix
| Profile | DB strategy | Flyway | Notes |
| --- | --- | --- | --- |
| `local-mysql` | `spring.jpa.hibernate.ddl-auto=update` | disabled for platform datasource | Fast local development, non-production policy |
| `local` | inherits base | enabled by base config | CORS local defaults |
| `staging`/`prod` | `ddl-auto=validate` expected | enabled | Migration-first policy enforced by startup validator |

## Canonical contract rules (effective immediately)
1. New backend features for active runtime must expose endpoints only in `/api/v1/*` or `/api/inventory/v1/*`.
2. New admin/storefront work must not depend on `legacy-storefront` profile routes.
3. Any legacy endpoint usage must include a migration plan and deprecation tag in docs.
4. API envelope and error code conventions must remain stable during Version 1.0.
